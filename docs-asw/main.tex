\section{Introduzione}
Il progetto HiHome consiste in un applicativo per gestire dispositivi all'interno di una smart home. Permette di gestire dispositivi smart da applicazione e aggiungere automazioni. Sono presenti funzionalità come notifiche, log di event relativi ai device smart, informazioni sullo stato attuale di temperatura, meteo e qualità dell'aria nella zona e grafici relativi ai valori passati di questi ultimi. Inoltre, è integrato con un agente AI in modo da effettuare alcune azioni in linguaggio naturale. L'applicazione supporta due tipi di utenti:

\begin{itemize}
    \item \textbf{Admin}: ha accesso a funzioni avanzate sulla gestione della casa, come log di eventi relativi a manipolazione di smart device, creazione di automazioni e aggiunta di device.
    \item \textbf{Utente Standard}: può agire sugli smart device, visualizzare statistiche relative allo stato attuale della casa, grafici relativi ad eventi passati, come lo stato della qualità dell'aria nella zona durante la settimana precedente e interagire con il chatbot.
\end{itemize}

\section{Requisiti}
\subsection{Requisiti funzionali}
\begin{itemize}
    \item Login
    \item Menu per navigazione tra le pagine 
    \item Dashboard con informazioni sullo stato attuale della casa e del circondario:
    \begin{itemize}
        \item Temperatura (interna ed esterna)
        \item Condizioni meteo attuali
        \item Qualità dell'aria
        \item Velocità e direzione del vento
    \end{itemize}
    \item Lista dispositivi smart, divisi per stanza, con possibilità di interagire con essi, in particolare:
    \begin{itemize}
        \item Accensione/spegnimento per luci
        \item Apertura/chiusura per finestre
        \item Impostazione temperatura per termostato
    \end{itemize}
    \item (Admin) Possibilità di aggiunta di nuovi dispositivi smart, con specifica del tipo di dispositivo, del nome e della stanza in cui si trova
    \item (Admin) Possibilità di creare automazioni, ognuna definita da:
    \begin{itemize}
        \item Nome
        \item Condizione di attivazione composta da un sensore, un operatore e un valore
        \item Una o più azioni da eseguire quando la condizione è verificata, ognuna composta da un dispositivo smart e un'azione da eseguire su di esso
    \end{itemize}
    \item (Admin) Possibilità di visualizzare la lista completa di automazioni presenti
    \item (Admin) Possibilità di visualizzare un log di eventi relativi alla manipolazione dei dispositivi smart, con specificate data, ora, dispositivo interessato, azione eseguita e utente che ha eseguito l'azione
    \item Grafici relativi a dati passati, in particolare:
    \begin{itemize}
        \item Grafico della temperatura esterna durante la settimana precedente, con temperatura massima e minima per ogni giorno
        \item Andamento della qualità dell'aria durante la settimana precedente
        \item Grafico di utilizzo dei dispositivi smart durante la settimana precedente, con numero di azioni per fascia oraria
        \item Dati relativi a numero di attivazioni manuali e automatiche per i dispositivi smart
        \item Dati relativi a percentuale di accensioni di luci e apertura di finestre
    \end{itemize}
    \item Chatbot con cui è possibile interagire in linguaggio naturale per eseguire azioni e chiedere informazioni a seconda dei privilegi dell'utente, ovvero:
    \begin{itemize}
        \item Aggiungere un dispositivo smart (Admin)
        \item Aggiungere una regola di automazione (Admin)
        \item Chiedere informazioni relative alle previsioni meteo per la settimana successiva
    \end{itemize}
    \item Notifiche relative a: 
    \begin{itemize}
        \item Esecuzione di un'automazione
        \item Livelli critici relativi alla qualità dell'aria
        \item (Admin) Azioni eseguite da utenti non admin
    \end{itemize}
    \item Impostazioni, con possibilità di:
    \begin{itemize}
        \item Effettuare logout
        \item (Admin) gestire gli utenti, con possibilità di visualizzare e modificare i permessi degli utenti della casa
        \item Definire quali notifiche ricevere
    \end{itemize}
    \item Piano giornaliero delle temperature, con possibilità di visualizzare la temperatura prevista per ogni ora del giorno e di impostare una temperatura desiderata per ogni fascia oraria
\end{itemize}
\subsection{Requisiti non funzionali}
\begin{itemize}
    \item Aggiornamento in tempo reale sullo stato dei dispositivi smart
    \item Interfaccia user-friendly e responsive
    \item Utilizzo di token per autenticazione della sessione
    \item Aggiornamento della chat con streaming delle risposte da parte del modello AI, per evitare tempi di attesa eccessivi
\end{itemize}
\subsection{Requisiti di implementazione}
\begin{itemize}
    \item Utilizzo di API RESTful per comunicazione tra frontend e backend
    \item Architettura MEVN (MongoDB, Express.js, Vue.js, Node.js), con aggiunta di un servizio esterno in Go per ottenere dati meteo e qualità dell'aria
    \item Utilizzo di Socket.io per notifiche, aggiornamenti in tempo reale e streaming delle risposte del modello AI
    \item Applicazione web responsive e mobile-first, con l'utilizzo di Tailwind CSS
    \item Utilizzo di Docker come tecnologia di containerizzazione per facilitare lo sviluppo e la distribuzione dell'applicazione
    \item Utilizzo di Semantic Versioning per il versionamento
    \item Git e Semantic Commit per la gestione del codice sorgente
\end{itemize}

\subsection{Design Architetturale}
Il progetto è stato realizzato seguendo la filosofia di design \textbf{Domain-Driven Design (DDD)}. Come pattern architetturale, è stata adottata un'architettura esagonale (ports and adapters) per ogni bounded context identificato. In particolare, sono stati identificati i seguenti bounded contexts:
\begin{itemize}
    \item \textbf{Home context}: è il contesto principale dell'applicazione, che si occupa della gestione delle case, dei dispositivi smart e della chat
    \item \textbf{User context}: si occupa della gestione degli utenti e dell'autenticazione
    \item \textbf{Notification context}: si occupa della gestione delle notifiche
    \item \textbf{Rule context}: si occupa della gestione delle regole di automazione
    \item \textbf{Weather context}: si occupa di ottenere dati relativi al meteo e alla qualità dell'aria da servizi esterni
\end{itemize}
L'architettura utilizza lo stack MEVN, con i primi 4 contesti implementati nel backend in Node.js, mentre il Weather context è implementato in un servizio esterno in Go.

L'applicazione HiHome è quindi divisa in diversi componenti. I client, che accedono all'applicazione tramite interfaccia web, il backend, che espone API RESTful per la comunicazione con i client e gestisce la business logic, e il servizio esterno per il meteo, che è interrogato periodicamente dal backend per ottenere dati aggiornati relativi alle condizioni nelle zone in cui le varie case si trovano. Il server backend, inoltre, accetta connessioni WebSocket dai client per l'invio di notifiche e aggiornamenti in tempo reale, e per lo streaming delle risposte del modello AI. Il backend accede ai dati tramite un database MongoDB.

Per quanto riguarda i concetti di dominio, si definiscono di seguito i principali, già brevemente descritti in precedenza:
\begin{itemize}
    \item \textbf{Casa}: rappresenta una casa intelligente, con i suoi dispositivi smart, le sue automazioni e i suoi utenti
    \item \textbf{Dispositivo Smart}: rappresenta un dispositivo intelligente all'interno di una casa, come una luce, una finestra o un termostato
    \item \textbf{Automazione}: rappresenta una regola di automazione, composta da una condizione di attivazione e da una o più azioni da eseguire quando la condizione è verificata
    \item \textbf{Utente}: rappresenta un utente dell'applicazione, con i suoi permessi e le sue preferenze
\end{itemize}
